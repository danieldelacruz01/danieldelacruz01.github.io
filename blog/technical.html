Testing

Explain to your non-tech friend

  • Scope: You use scope to set which functions have access to which variables and how long they are valid for. In JavaScript, there two scopes (global and local). Local variables are variables which are declared within JS functions and are therefore only accessible within those functions. The variable becomes invalid once the function has finished running. On the other hand, global variables are declared outside of functions and can be accessed and modified by all functions in the code. The variable stays valid as long as the page is open.
  • Parentheses (): Parentheses are used to hold arguments when calling functions. They can also be used to set the order of operations, just like in algebra.
  • Braces {}: The curly braces are used to set the start and end of of a function. When the function is called, all code within the braces will run.
  • Square brackets []: Square brackets are used to contain or access values of an array. An array allows you to store multiple values in a single variable by using sqaure brackets and separating each value with a comma. (e.g. var myArray = [0,1,2,3,4,5] ). You can also use square brackets to access individual values in the array (e.g. myArray[0] returns the first value in the array).
  • Single quotes ' ' and double quotes " " : Both single quotes and double quotes are used to contain strings (e.g. "My name is Daniel"). Either of them are acceptable, although you need to be careful not to include the same type of quotes within quotes (i.e. "His name is "Johnny"" will be shortened to "His name is ", but "His name is 'Johnny'" will work fine.

Explain to your non-tech friend

  • .map(): This allows you to create a new array using a function that is called on each item of an existing array. For example if you had an array of numbers, you could create a function that multiplies the value by two, then use .map() to create a new array that doubles each value in the exisitng array by calling the multiply function on each item within it.
  • .filter(): this allows you to test all items in an array aginst a condition, and create a new array that returns all items that either meet or don't meet that condition. A practical application for this could be filtering all items in an online shop above a certian price.
  • .reduce(): I still don't fully grasp how .reduce() works, but from what I understand it reduces all the values in an array to into just one value. For example, you would use this if you wanted to return the sum of all numbers in an array. It does this but adding the first two items in the array together, then adding the output to the next item in the array until it reaches the end.

Recursion vs Iteration

Both recursion and iteration are used to repeat functions over and over, but they do so in a slightly different ways. Iteration is basically just a loop of the same function over and over again, whereas recursion is like a nesting function that calls itself (kind of like function-inception!). This explanation uses factorials as an example of recursion (i.e. x(x-1)(x-2)(x-3)... until 1 is reached).

JavaScript Basics

How does Javascript compare to HTML and CSS?

  • HTML is used to establish the structural elements (or content) of a webpage. It tells the browser which elements are headings, which elements are paragraphs, which elements are images, for example.
  • CSS defines how the HTML elements look on the webpage. For example, you might use CSS to tell the browser that paragraphs should be displayed in a certain typeface like Arial, or that images should have yellow borders.
  • JavaScript is used to create functions that alter the way a site looks and behaves in response to user inputs. For example, you would use JavaScript to create a menu that drops down in response to you clicking on it, or hovering your mouse over it. JS essentially makes the website more interactive.

Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'

Control flow and loops are used to determine the order in which a program runs its code. They include statements like 'if (this condition is met) {perform this action} else {perform this action}' or 'while (this condition is true) {perform this action}'. We actually use similar control flow and loops in our own everyday lives. For example, when deciding whether or not to take a pizza out of the oven, you might think to yourself 'if (pizza = hot) {remove from oven} else {leave in oven}.

Explain the difference between accessing data from arrays and object literals

Both arrays and objects are used to store data, but the ways they are stored, accessed, and edited are very different. Arrays are useful for data that can be stored in an ordered list. For example, storing the results of a horse race from first place to last place. Objects allow you to store and access your data in a more methodical way. Consider Netflix as an example. It would be a nightmare to have every single film and TV show on Netflix in one long list. It makes a lot more sense to split them into categories of 'movies' and 'TV shows'. You could then create subacategories according to genre, like 'comedy', 'drama', 'action', etc. Objects store data (and arrays of data) in a similar way.

Explain what functions are and why they are useful

A function is like a snippet of code that can be used and re-used as many times as you like using different parameters. For example, say you wanted to multiply two numbers together. You could create a function called 'multiply' that uses X and Y as parameters, and runs some code that multiplies X and Y together. This is how it would look:

var multiply = function (X, Y) {
console.log (X * Y);
}

Then, when you run the function by entering something like 'multiply (4, 5)', the console would multiply the two parameters together and display the answer (20)

Functions are useful because you can re-use them as many times as you like, so you don't have to write the code you would like to execute out manually every time.

Design-to-Web

What is a responsive site, and why is responsiveness important?

Reponsive websites are websites that adapt their content to fit your device's screen size and resolution. This is so that the content makes optimal use of the space on the screen of whichever device you're accessing the website from. It is extremely important in this day and age in which screens range in size from 3.5 to 30+ inches, and users are accessing websites across a range of different devices, including phones, tablets, laptops, etc. Responsive design helps ensure that the user experience and website design language is consistent across all devices.

What is mobile-first design and why is it important?

As the name implies, mobile-first design is designing a website with the mobile layout in mind first and foremost. The rationale behind this is that it is simple to convert a mobile layout into a desktop layout because of the increased screen real estate. Designing a desktop site first, then attempting to convert it into a mobile layout is much more difficult as the designer has to work out how to cram all the webpage elements in a smaller form factor. The analogy of moving house is a good one to describe why mobile first design is important; it is very easy to move from a small apartment into a big house, but it is difficult to move from a large house with lots of furniture into a small apartment.

What are frameworks? What are their pros and cons?

Frameworks are standardised and free-to-use bit of code designed to be reused across many websites and web pages. Their purpose is generally to solve a common problem in web design (e.g. how to create a responsive site). For example, this website uses the Skeleton CSS framework for reponsive design. Skeleton is basically a stylesheet that tells the website how to look depending on the user's screen size. Instead of the developer needing to code it from scratch, they can simply use a framework like Skeleton as a starting point.

Pros

  • Time-savings
  • Consistency
  • Tidier code
  • Reduced human error

Cons

  • Prevents you from learning it yourself
  • Your own customisations can potentially conflict with the framework, so it can therefore be quite limiting

What are wireframes and why do we use them?

Wireframes are basic sketches of the bare-bones structure & layout of the website. They help the developer visualise what the final product will look like, without the need for actually coding it. The wireframes can then be translated into a boxified design.

Udacity Reflection pt. 2

What is grid based design?

Grid-based design is a way to split a webpage up into a an invisible grid in order to create the layout of the page more easily. A grid of any number of rows and 12 columns is common as it enables content on a page to be arranged in grids of 1, 2, 3, 4, 6, and 12. If we use percentages to specify the size of the columns instead of pixels, the page can be resized more easily and accommodate a range of screens at different resolutions.

What's all the hype about responsive webpages?

More and more people are accessing the internet from mobile devices such as phones or tablets with a broad range of screen sizes and resolutions. When smartphones and tablets first came out, it was common to see separate mobile sites that would function a bit differently or not have as many features as desktop sites. Responsive design allows the user to view the same webpage across devices with the content adapting to fit the screen they are viewing it from. This enables a consistent user experience and design language across devices.

What is semantic structure?

Semantic structure is defined by the conventions surrounding how programmers structure their HTML. While a browser might be able to read HTML that is on a single line, following semantic structure guidelines make it a lot more comphrehensible to humans. For example, a human would be able to look at an h1 tag and know that the content is most likely an important heading at the top of the page, whereas h2 or h3 text would most likely be sub-headings that are of lesser relative importance than the h1 tag.

Other thoughts

It was really cool seeing reponsive design in action on the Gizmodo (US) site by observing how the layout changes as you adjust the width of the browser window. Another page that I think does responsive design really well is the Flick electric website.

HTML, CSS, and the DOM: A non-technical analogy

I'm still trying to get my head around the concept of the DOM (Direct Object Model) and how it relates to HTML and CSS. If my understanding is correct, I think the different elements of a short story might make a good analogy for how they interrelate:

HTML elements are like the subjects and objects in a short story (i.e. all the nouns). Without them the story doesn't really exist.

CSS can be likened to the descriptive words used (i.e. adjectives, adverbs) to give the story character and bring it to life.

(If I understood correctly) the DOM is kind of like the reader's understanding of language. Much like how we use our knowledge to determine which words are subjects/objects, and which are descriptive words, our web browsers use the DOM read and interpret nodes on a webpage.

What is boxifying design?

Boxifying design is the process of taking a design mockup and creating a very basic 'wireframe' of the webpage layout by physically or mentally drawing boxes around the different elements & sections of the page.

What is the Box Model?

The box model is a design rule that describes each element of a webpage as a box (regardless of its actual shape). The box is made up of several visible and invisible layers; the margin, the border, the padding, and the content itself.
